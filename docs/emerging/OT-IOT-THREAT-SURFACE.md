# OT / IoT Threat Surface Notes (adjacent thinking)

SecureSkies is **not** an industrial OT deployment. These notes capture transferable questions when drone/robotics concepts approach IoT/OT-like constraints.

## Surface categories

1. **Sensors** — spoofing / jamming awareness (conceptual)
2. **Links** — command and telemetry channels
3. **Ground station** — operator workstation compromise
4. **Updates** — firmware/software supply chain for companions
5. **Data at rest** — logs and mission artifacts sensitivity

## Defensive posture (learning)

- Inventory interfaces before expanding features
- Prefer least privilege on ground tools
- Keep private academic media out of public repos

## Non-claims

Not an IEC/ISA or sector OT compliance package.
