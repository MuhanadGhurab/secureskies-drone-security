# Security Policy — SecureSkies Public Repository

## Scope

This repository is **documentation-focused**. It preserves a sanitized historical
portfolio of the SecureSkies university project and plans modern simulation work.

It does **not** include:

- Real drone control code
- Active network scanning
- Live surveillance capability
- Credentials or secrets
- Live telemetry streams
- Physical safety or airworthiness claims
- Regulatory approval claims
- Production security product claims
- The private academic PDF or student identifiers

## Responsible reporting

If you discover a security issue in this documentation repository (for example: an
accidentally committed secret, sensitive pattern, or unsafe recommendation):

1. Do **not** open a public issue with secrets
2. Email: **muhanadghurab@gmail.com**
3. Allow a reasonable time for remediation before public discussion

## Private academic material

Do not request redistribution of the original academic report, student identifiers,
signature pages, maps, or unapproved photographs through this repository.

## Future digital twin

Any future software digital twin (planned separately) must remain simulation-only:
no real aircraft actuation, no live camera feeds, and no operational surveillance.
